纽约出租车出行数据分析业务报告
一、项目概述
本报告基于 Kaggle 公开纽约出租车百万级出行数据集，通过完整的数据清洗、多维特征构建、异常样本识别、行程时长预测、用户分群挖掘、多维度指标统计，挖掘出行客流、订单异常、用户行为规律，输出网约车平台可落地运力、风控、营销优化策略。
原始数据集 Trip_Duration_Data.csv 体量超 10GB，本地无法完整加载，本次分析使用经过标准化清洗编码后的 data_encoded.csv 子集开展全流程挖掘。
二、数据预处理结果
2.1 基础数据校验
缺失值检测：数据集全部字段无空值，无需填充处理；
重复样本检测：存在少量完全重复订单，已全部剔除，避免数据重复干扰分析结果；
异常数值校验：乘客人数存在 0 人、7 人以上无效订单，结合 IQR 四分位阈值划定合理区间 1~6 人，过滤异常样本。
2.2 脏数据过滤规则
剔除乘车人数不在 1~6 区间的订单；
过滤超出纽约标准经纬度范围的坐标脏数据；
清除车速无穷大、无行驶距离的无效行程记录；
采用 IQR 缩尾处理方式约束时长、距离极端值，不直接删除样本，保留完整订单基数。
2.3 特征工程产出
时序衍生特征：提取订单上车小时、星期、月份，区分高峰 / 平峰、工作日 / 周末、季节出行差异；
空间距离特征：基于经纬度计算哈弗辛球面距离、曼哈顿距离、欧式平面距离，精准还原实际行驶里程；
业务衍生特征：结合行程时长与行驶距离计算平均车速，作为识别虚假行程核心指标；
标签变换：对右偏分布的行程时长做 log1p 对数变换，降低长尾数据对模型预测的干扰。
三、EDA 探索性分析结论
3.1 特征相关性分析
行驶距离与行程时长呈现强正相关，是预测时长的核心变量；
三类距离特征之间相关系数超过 0.8，存在明显多重共线性，建模时按需筛选其一即可；
上车小时、星期与行程时长存在弱相关，证明时段路况会影响通行耗时。
3.2 距离与车速分布
原始数据存在大量超长距离、极端低速 / 高速异常订单；经 IQR 缩尾处理后，数据分布趋于稳定，绝大多数订单集中在短途日常出行区间。
3.3 分时段客流规律
全天存在两段明显出行高峰：早 7:00-9:00、晚 17:00-20:00，两段高峰订单总量占全天 40% 以上；午间、凌晨订单量显著偏低，属于运力闲置平峰期。
3.4 服务商运营差异
不同服务商承接订单总量、平均行驶距离、平均行程时长存在区分，部分服务商平均耗时更长，路线规划、司机运营效率存在优化空间。
四、异常订单识别分析
使用 IsolationForest 孤立森林无监督模型，基于标准化距离、车速、时长特征识别异常样本：
异常订单特征：远距离夜间行程、车速极端、坐标偏移；
业务风险：此类订单大概率存在绕路虚报、虚假下单问题，会提升平台赔付成本；
处理建议：长途、夜间订单开启人工复核通道，降低平台资金损失。
五、LightGBM 行程时长预测模型分析
5.1 模型训练方案
采用 8:2 划分训练集与验证集，搭配 5 折交叉验证、网格搜索完成超参调优，使用 L1、L2 双重正则缓解过拟合。
5.2 模型评估指标
以原始秒数作为真实业务单位评估：
验证集 MAE：预测与真实时长平均误差可控；
验证集 RMSE：整体预测偏差稳定，模型具备实用预估能力。
5.3 核心影响特征排序
球面行驶距离、上车时段、平均车速为影响打车时长 Top3 核心特征，其余月份、服务商权重占比较低。
六、K-Means 用户分层分析（自主拓展模块）
基于行驶距离、行程时长、上车小时三类行为特征完成聚类，通过肘部法则验证最优分群数量为 3 类：
通勤短途用户
集中早晚高峰出行，单次行驶距离短、耗时少，平台高频基础用户；
夜间长途游客用户
多晚间下单，跨区域远距离行程，订单单价高，异常风险同步升高；
午间休闲用户
工作日午间零散出行，出行距离中等，订单频次偏低。
七、多维度运营指标统计（Python+MySQL 联动）
通过分组聚合、窗口函数完成服务商、时段、工作日多维度报表统计：
各服务商总订单、平均行驶距离、平均通行时长对比；
24 小时分时段订单数量、客流占比、高峰排名；
工作日与周末订单总量拆分，区分通勤与休闲出行需求差异。
八、平台落地运营优化方案
8.1 运力调度优化
早、晚两大客流高峰，在热门上下车商圈增派在线司机，缩短乘客等待时长；
午间、凌晨平峰期下调在线运力，减少司机空驶里程，降低平台运维成本；
冬季雨雪月份整体行程时长偏高，高峰适度追加备用运力。
8.2 订单风控优化
夜间远距离订单系统自动标记，触发人工复核流程，拦截绕路虚假行程；
对车速极端、坐标异常的行程实时预警，提前介入核查；
定期统计各服务商异常订单占比，对高风险服务商开展司机规范培训。
8.3 用户分层精细化营销
通勤短途群体：早晚高峰发放小额通勤优惠券，提升日常复购频次；
夜间长途游客：推出长途出行打包优惠套餐，兼顾营收与风控校验；
午间休闲群体：工作日中午限时折扣，拉动平峰闲置运力利用率。
8.4 服务商运营管控
定期对比各服务商平均行程耗时、异常订单占比，针对运营效率偏低的服务商统一开展路线规划、行车规范培训，整体提升平台出行体验。
九、总结
本项目完成从原始脏数据处理、特征挖掘、异常识别、时长预测、用户分群全链路数据分析，完整挖掘网约车客流、风险、用户三大核心业务规律，全部分析结论可直接用于平台运力调配、订单风控、分层营销落地，具备真实商业使用价值。
